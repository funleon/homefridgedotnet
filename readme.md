# 冰箱食品管理 Web 系統 (HomeFridgeDotNet)

本專案是一個使用 **.NET Core 8 MVC** 架構開發的 Web 應用程式，專門設計用於協助使用者有效管理與追蹤冰箱內的食品項目。使用者可以輕鬆地新增、編輯、刪除和查詢食品，並詳細記錄食品的到期日、儲存位置、備註以及使用進度等資訊。

## 🎯 核心功能

*   **食品列表展示**: 清晰顯示所有記錄在冰箱中的食品詳細資訊。
*   **新增食品追蹤**: 允許自訂輸入食品名稱、數量、到期日、儲存位置（如：冷藏區、冷凍庫）及其他備註。
*   **編輯與管理**:
    *   隨時更新現有食品的狀態與資訊。
    *   **「已使用%」進度追蹤**: 提供 0 到 100 的整數百分比輸入，直觀顯示該食品剩餘與消耗的狀況。
*   **安全移除**: 支援從系統記錄中永久刪除已使用完畢或過期的食品。
*   **便利查詢**: 支援多維度搜尋，可依據食品名稱、到期日或儲存位置等條件快速過濾食品列表。

## 🛠️ 技術棧與架構設計

本系統遵循標準的 .NET Core MVC 架構模式，並透過服務層與資料存取層的拆分來實現高內聚、低耦合的設計。

*   **後端框架**: ASP.NET Core 8 MVC
*   **前端介面**: HTML5, CSS3, JavaScript (結合 Razor 視圖引擎 `*.cshtml`)
*   **資料儲存**: 輕量化 **INI 文字檔案** (`foods.ini`)
    *   每個食品項目以唯一的 Section (如 `Food_GUID`) 進行儲存。
    *   資料讀寫邏輯集中管理於資料存取層 (Data Access Layer)。

### 專案結構概覽

*   **`Controllers/`**: 負責接收 HTTP 請求，協調 Models 與 Views 的互動。保持輕量，專注於流程控制。
*   **`Services/`**: 系統核心業務邏輯的封裝層，處理資料操作前的業務驗證，並與前方的 Controller 及後方的資料庫交互。
*   **`Models/`**: 定義系統中的資料實體結構（例如 `FoodItem`）。
*   **`Views/`**: 包含所有呈現給使用者的 Razor 介面檔案，依功能模組化劃分。
*   **`Data/`**: 專門處理與 `foods.ini` 檔案的實際反序列化與寫入操作。

## 🚀 系統安裝與環境設定

請依照以下步驟在本地環境中執行本系統：

1.  **環境需求**: 確認您的電腦已安裝相應版本的 [.NET Core 8 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)。
2.  **取得專案**: 從原始碼儲存庫 (GitHub) 複製或下載本專案至本地資料夾。
3.  **基礎設定**:
    開啟應用程式設定檔 `appsettings.json`，確保 `IniFilePath` 的設定值指向正確且可供程式讀寫的 `.ini` 檔案路徑（預設名稱建議為 `foods.ini`）。
    ```json
    {
      "IniFilePath": "您的/本地/路徑/foods.ini"
    }
    ```
4.  **開啟專案**: 您可以使用 **Visual Studio 2022** 開啟 `.sln` 方案檔，或是使用輕量級的編輯器（如 VS Code）配合 **.NET CLI**。
5.  **編譯系統**: 在終端機/命令列中導覽至專案根目錄（包含 `.csproj` 的資料夾），並執行以下命令：
    ```bash
    dotnet build
    ```

## 🏃 執行方式

1.  在您的 IDE 中按下執行鍵（如 Visual Studio 的 IIS Express / HomeFridgeDotNet 設定檔），或在命令列執行目錄為專案資料夾下的動作：
    ```bash
    dotnet run
    ```
2.  編譯並啟動伺服器後，系統會於終端中提示伺服器運作中的 localhost 網址與埠號。
3.  在瀏覽器中開啟 `https://localhost:<port>` 或是 `http://localhost:<port>` （依據實際組態）。
4.  順利進入首頁後，即可開始建立您的第一筆冰箱食品紀錄！
